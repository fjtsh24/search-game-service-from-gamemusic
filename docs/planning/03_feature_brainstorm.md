# 03 機能ブレスト

## 評価記号の凡例（ご自由にお使いください）

| 記号 | 意味 |
|------|------|
| ◎ | ぜひ欲しい・外せない |
| ○ | あると良い |
| △ | 面白いが優先度は低め / 要検討 |
| ✗ | 不要・方向性と合わない |
| ? | もっと議論したい |

## 領域①：探索・発見（コア）

| # | 機能名 | 概要 | 評価 | コメント |
|---|--------|------|------|---------|
| 1-1 | ゲーム起点の類似ゲーム表示 | ゲームを入力すると、音楽的に似たゲームを一覧表示する |◎|ただし、単一ゲームからの候補の表示ではなく、複数のゲームの好みの傾向から音楽的に似たゲームを一覧表示したいです|
| 1-2 | アーティスト/作曲家起点の探索 | アーティスト・作曲家名を入力すると、その人が手がけたゲーム一覧を表示する |◎| |
| 1-3 | 雰囲気タグによる絞り込み | 「ダーク」「ポップ」「環境音楽的」「壮大なオーケストラ」などのタグでゲームを絞り込む |◎| |
| 1-4 | 類似ゲームのグラフ表示 | Last.fmの「Similar Artists」に相当する、ゲーム間の音楽的類似度をグラフ（ネットワーク図）で可視化する |△| |
| 1-5 | シーン/曲単位での検索 | 「このゲームのこのシーンの曲のような雰囲気」という細かい指定から探す |△| |
| 1-6 | ラジオモード | 好きなゲームを起点に、次々とゲームを自動提案してくれるラジオ的な探索モード |○|ゲームのサントラをユーザの好みの傾向を元に自動でランダム再生できてそこからゲームの概要ページに飛べるのが理想|

## 領域②：フィードバック・学習ループ

| # | 機能名 | 概要 | 評価 | コメント |

| 2-3 | ウィッシュリスト保存 | 気になったゲームをリストに保存する |✗|steamの機能でしてもらえば良いので、steamページに飛ばせて後はそっちでお願いしたい|
| 2-4 | レコメンド理由の表示 | 「〇〇が好きな人に人気」「〇〇と同じ作曲家」のようにレコメンドの理由を表示する |△|作曲家は示したいかも|

## 議論ポイントの解決記録

| # | 問い | 決定 |
|---|------|------|
| Q1 | 1-1と4-3は別物か | **同一エンジン・表示場所違い**として統合。ゲームページ上の「似たゲーム欄」とホームの「おすすめフィード」は同じレコメンドロジックで動く |
| Q2 | Steam以外のプラットフォームはどう扱うか | **MVP時点ではSteamのみ対応**。他プラットフォームはOSS拡張として外部貢献に委ねる（OSSとして公開する想定） |
| Q3 | 音楽再生手段 | **YouTube優先**（アカウント登録不要・埋め込み再生可）。Spotify は補助的な選択肢 → 1-6ラジオモードのMVP昇格を検討 |
| Q4 | Last.fm連携 | **スコープ外に確定**（方向性が不明確なため除外） |
