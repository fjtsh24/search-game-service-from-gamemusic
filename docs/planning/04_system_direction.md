# 04 システム方向性 統合提案

5エージェント（OSS戦略・データ設計・セキュリティ・技術スタック・統合）による分析結果。2026-07-18 策定。

---

## 1. OSS公開判断 — 段階的公開（AGPL-3.0）

**設計はOSS化を前提に、リポジトリ公開はMVP安定後。** この二つは独立した判断。

- **フェーズ1（開発初期）**: 非公開で設計を自由に変える。ただし `integrations/steam.py` のようなプラグイン層の分離設計だけは最初から行う
- **フェーズ2（MVP安定後）**: ドキュメントを整備してAGPL-3.0で公開。「対応は保証しない趣味プロジェクト」と最初から明示する
- **OSS化の前提条件**: すべてのAPIキーを環境変数化し、各セルフホスト利用者が自分のキーを使う設計にしておくこと。後回しにすると大規模リファクタリングが必要になる

## 2. 推奨技術スタック

| 層 | 採用技術 | 採用理由の核心 |
|---|---|---|
| フロント | Next.js 15 + TypeScript | ラジオモードのグローバルプレーヤーをApp Routerで自然に実装 |
| バック | Python + FastAPI | 将来のML音楽分析が同一言語で完結。Swagger UIがOSS参入障壁を下げる |
| DB | PostgreSQL（Supabase） | pgvectorで将来のベクトル検索を同一DBに統合。RLSで最小権限原則を強制 |
| キャッシュ | Upstash Redis | 管理ゼロのサーバーレス課金。APIレートリミット管理に活用 |
| ホスティング | Vercel + Fly.io | git pushで自動デプロイ。月$0〜$5 |

**採用しない技術**: Elasticsearch・GraphQL・SvelteKit・自前Redisホスト。スケール問題が実際に起きてから追加する。

## 3. データ戦略

**「コアデータは自前保有、揮発性データはキャッシュで吸収」** が原則。

| データ | 戦略 |
|-------|------|
| VGMdb（ゲーム×曲×作曲家） | 初期インポートで完全自前化し依存を断つ |
| YouTube VideoID | 事前バッチ検索してDB保存。埋め込み再生はクォータ不消費 |
| Steam API | ユーザー操作トリガーのみ。定期ポーリング禁止 |
| Spotify | 依存を最小化（25ユーザー制限の問題あり）。Last.fmで代替できる部分は代替 |

### 音楽類似度の計算方法（MVP）

**VibeTag Jaccard係数** を採用。「なぜ似ているか」をタグで説明できる解釈可能性が、このサービスのコンセプト（気付きを与える）と合致する。

### データ品質方針

**Graceful Degradation**: データ欠損はエラーではなく状態として扱い、70%の品質でリリースする。

## 4. セキュリティ設計

### MVP必須（先送り不可）

| # | 内容 | 理由 |
|---|------|------|
| 1 | **Steam OpenID 2.0 の正確な実装** | OAuthではない。`check_authentication` の省略がなりすましに直結する |
| 2 | **環境変数3点セット** | `.env` + `.gitignore` + `.env.example`。GitHub Secret Scanningも有効化 |
| 3 | **Supabase RLS** | DBレベルでアクセス制御を強制。アプリバグがデータ漏洩に直結するリスクを排除 |
| 4 | **アカウント削除フロー** | 後回しにするとコストが著しく高くなる。初期に設計する |

### セッション vs JWT → **セッション採用**

JWTは採用しない。サーバーサイドセッションのほうがMVPには適切（即時無効化可能、実装シンプル）。

## 5. 専門家意見の分岐点と判断

| 論点 | 判断 | 理由 |
|---|---|---|
| セッション vs JWT | セッション採用 | 複雑性が1人開発のMVPに見合わない |
| Spotify依存度 | 最小化 | 25ユーザー制限がOSS化と相性が悪い |
| OSS公開タイミング | 設計先行・公開後回し | 設計はOSS前提に、リポジトリ公開はMVP後 |
| データ完全性 vs 速度 | 速度優先 | Graceful Degradationで欠損を許容 |
| モノレポ初期導入 | MVP後回し | セットアップコストより動くものを優先 |
