# 05 ユーザーストーリーマップ

## ウォーキングスケルトン（最小体験フロー）

サービスの価値を証明できる最小の通り道。MVPはこの2本を完走できることを最初のゴールとする。

### Path A：ペルソナA（音楽起点の探索者）

```
アーティスト名で検索
  → アーティストページ（担当ゲーム一覧）
    → ゲーム詳細ページ（タグ・類似ゲーム）
      → YouTubeでサントラ試聴
        → Steamページへ
```

### Path B：ペルソナB（無自覚な音楽感受性ゲーマー）

```
Steamアカウントでログイン
  → ライブラリをインポート
    → プレイ済みゲームに星評価
      → パーソナライズドフィードでレコメンド表示
        → ゲーム詳細ページ
          → Steamページへ
```

---

## ストーリーマップ

### 活動①：入口（Discovery Entry）

**MVP**
- アーティスト・作曲家名でテキスト検索する
- ゲーム名でテキスト検索する
- 雰囲気タグ一覧から絞り込む
- ログイン後のホームフィードを見る

**V2**
- ラジオモードを起動する

---

### 活動②：探索（Explore）

**MVP**
- アーティストページでそのアーティストの担当ゲーム一覧を見る
- タグ別のゲーム一覧を見る

**V2**
- ゲーム間の音楽的類似グラフを辿る

---

### 活動③：ゲーム詳細（Game Detail）

**MVP**
- ゲームのタイトル・説明・雰囲気タグを確認する
- 担当作曲家・アーティストを確認する
- 音楽的に似たゲームの一覧を同ページで見る

**V2**
- レコメンドの理由（「〇〇と同じ作曲家」など）を確認する

---

### 活動④：音楽試聴（Music Preview）

**MVP**
- ゲームのサントラをYouTube埋め込みで試聴する

**V2**
- ラジオモードで好みの傾向に合った曲が自動再生される
- 再生中の曲のゲームページへ遷移する

---

### 活動⑤：好み記録（Preference Logging）

**MVP**
- Steamアカウントでログイン・アカウント作成する
- Steamライブラリをインポートする
- プレイ済みゲームに星評価（1〜5）をつける

---

### 活動⑥：レコメンド受け取り（Recommendation）

**MVP**
- パーソナライズドフィードで未体験ゲームのレコメンドを受け取る
- 星評価を追加するとフィードが更新される

**V2**
- レコメンド理由を確認する

---

### 活動⑦：行動（Action）

**MVP**
- ゲームのSteamページへ飛ぶ
- フィード上のゲームをSteamで確認する

---

## MVPの優先順位（開発順序の目安）

| フェーズ | 内容 | 証明できること |
|---------|------|--------------|
| **M1: データ基盤** | Steam人気サントラ200件を取込み、作曲家・タグ・YouTubeIDを整備する | 音楽グラフが動く |
| **M2: 検索・閲覧** | アーティスト検索→ゲーム詳細→YouTube試聴 | Path Aが通る |
| **M3: 認証・連携** | Steam OpenID → ライブラリインポート → 星評価 | Path Bの前半 |
| **M4: レコメンド** | 星評価を元にしたパーソナライズドフィード | Path Bが完走する |

> **注**: タグ絞り込み（旧M5）はM1のデータ整備（`game_tags`投入）と同時に機能する。M2のAPIとUIは実装済みのため、データが入れば即使用可能。
